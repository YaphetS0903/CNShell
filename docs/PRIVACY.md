# 隐私说明

CNshell 默认不上传遥测、连接资料、终端内容、命令历史或崩溃信息。连接配置保存在本机应用数据目录，秘密保存在 macOS Keychain 或 Windows 凭据管理器。远程通信只发往用户配置的服务器、代理、跳板机、WebDAV/AI Provider 或团队 relay；这些联网能力均由用户明确配置。检查更新只访问发行配置中的 CNshell HTTPS 更新端点。

加密同步口令默认不保存。用户明确启用生物识别保护后，macOS 使用仅限本机、绑定当前指纹集合的 Data Protection Keychain 项；Windows 使用 Windows Hello 保护的 CNG 非导出密钥封装口令。凭据账户标识来自同步目录规范化路径的 SHA-256，不包含原路径。解锁后的口令只交给本机 Rust 同步函数，不返回前端、不写入数据库或诊断；用户可随时移除，并继续使用手动口令。

用户可删除连接、清空应用数据或卸载应用。连接移入“已删除项目”时保留系统凭据以支持恢复；永久删除会先清理对应 macOS Keychain 或 Windows 凭据管理器条目。若系统凭据库拒绝清理，CNshell 会保留可重试的本地配置并明确报错；后续数据库操作失败时会尽力恢复原凭据。

“使用默认应用打开”会在系统临时目录创建远程文件副本，供所选本地应用读取。CNshell 每次启动时尽力清理上次运行遗留的预览缓存；如内容高度敏感，请在使用后关闭本地预览应用并重新启动 CNshell。
