# 隐私说明

CNshell 默认不上传遥测、连接资料、终端内容、命令历史或崩溃信息。连接配置保存在本机应用数据目录，秘密保存在 macOS Keychain。远程通信只发往用户配置的服务器、代理和跳板机；检查更新只访问配置的 CNshell 更新端点。

加密同步口令默认不保存。用户选择“用 Touch ID 保存”后，CNshell 只在这台 Mac 的 Data Protection Keychain 中保存口令；Keychain account 是同步目录规范化路径的 SHA-256，不包含原路径。Touch ID 解锁后的口令只交给本机 Rust 同步函数，不返回前端、不写入数据库或诊断；用户可随时移除，并继续使用手动口令。

用户可删除连接、清空应用数据或卸载应用。连接移入废纸篓时保留 Keychain 凭据以支持恢复；永久删除会先清理对应凭据，若 macOS Keychain 拒绝清理则保留可重试的本地配置并明确报错。后续数据库操作失败时会尽力恢复原凭据。

“使用默认应用打开”会在系统临时目录创建远程文件副本，供所选本地应用读取。CNshell 每次启动时尽力清理上次运行遗留的预览缓存；如内容高度敏感，请在使用后关闭本地预览应用并重新启动 CNshell。
