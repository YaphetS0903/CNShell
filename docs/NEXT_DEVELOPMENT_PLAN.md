# CNshell 后续功能开发总规划

> 文档状态：v1.4 执行中（阶段 1–4 核心实现已完成，外部硬件/真机验收待补）
>
> 制定日期：2026-07-14
>
> 适用范围：当前候选版之后尚未开发、未完整开发或尚未完成正式验收的能力

## 1. 目标与原则

本规划覆盖 Zmodem、Mosh、X11、RDP 内嵌、高级密钥、扩展自动化、AI、插件与团队协作、遗留协议、内置云同步客户端以及正式发布能力。已有 SSH、SFTP、监控、端口转发、受限自动化和加密目录同步继续作为稳定基线。

实施原则：

1. 每个阶段形成可独立使用、测试和回滚的交付物，不同时铺开全部功能。
2. 密码、私钥、终端内容和连接资料默认只留在本机；任何上传必须由用户明确选择范围。
3. 协议能力必须真实实现并完成互操作验收，检测到依赖不等于功能完成。
4. 高风险操作必须预览、确认、可取消并记录本机审计事件。
5. 第三方组件必须固定版本和哈希、审计许可证、构建 universal binary 并纳入签名。
6. 长测、1 GB 文件和跨设备验收只在阶段完成时执行，不纳入每次普通开发门禁。

## 3. 阶段 0：正式发布与权限基线

### 范围

- 使用 Developer ID Application 为主程序和所有 sidecar 签名。
- 完成 Apple 公证、票据装订和 Gatekeeper 验证。
- 配置 Tauri 正式更新公钥、HTTPS endpoint、签名归档和回滚策略。
- 确定 App Sandbox、Hardened Runtime、网络、文件、自动化和辅助功能权限策略。
- 建立崩溃诊断的本地导出流程；默认不自动上传遥测。
- 在干净 Mac 上验证安装、首次启动、升级、卸载和数据保留。

### 验收标准

- `spctl --assess`、`codesign --verify --deep --strict` 和 `stapler validate` 全部通过。
- 更新失败时保留当前可运行版本和用户数据。
- 更新服务不可用、签名错误或版本降级时明确拒绝安装。
- Ventura、Sonoma、Sequoia、Apple Silicon 和 Intel 至少各有一次真实运行证据。

### 外部输入

- Apple Developer Program 账号、Developer ID 证书和公证凭据。
- Tauri updater 密钥对和稳定 HTTPS 发布地址。
- Intel Mac、不同 macOS 版本设备或可信云 Mac 测试资源。

## 4. 阶段 1：Zmodem 完整传输

### 当前进度（2026-07-15）

- 已完成 SSH 原始字节流握手检测、CRC 校验、协议帧接管和日志隔离。
- 已完成 macOS 原生上传/下载授权、多文件、空文件、中文空格文件名、自动冲突改名、原子下载、取消、授权超时、断线恢复和实时进度 UI。
- 已与腾讯云 Linux x86_64 `lrzsz` 完成真实双向小文件及多文件互操作并校验远端退出状态。
- 已通过 Rust 全量测试、前端全量测试和 arm64 应用打包安装；100 MB、磁盘满和人工中途取消作为阶段验收专项，不在普通迭代中重复执行。

### 范围

- 在 SSH 原始字节流中识别 `rz/sz` 握手序列，不把协议字节写入 xterm。
- 建立每会话状态机：空闲、协商、上传、下载、取消、成功和失败。
- 上传和下载都通过 macOS 原生文件选择器授权。
- 支持单文件、多文件、进度、速度、取消、超时、重试和冲突策略。
- 传输时暂停普通终端输入，结束后恢复同一 PTY。
- 对文件名、路径、大小和协议帧设置上限，拒绝远端任意覆盖本地路径。

### 技术方案

- Rust 侧接管 SSH channel 原始输入/输出并运行 Zmodem 编解码器。
- 前端仅接收传输状态事件，不解析二进制协议。
- 优先评估许可证兼容的 Rust 实现；若引入 sidecar，固定源码哈希并分发许可。
- 复用现有传输队列，但与 SFTP/SCP 任务明确区分协议类型。

### 验收标准

- 与 Linux `lrzsz` 完成上传、下载、多文件、空文件和中文空格文件名互操作。
- 100 MB 文件 SHA-256 一致；取消不暴露为完整文件。
- 握手误识别不会吞掉普通终端输出。
- 断线、超时、磁盘满和远端拒绝均能恢复终端并给出可操作错误。

## 5. 阶段 2：Mosh 漫游连接

### 当前进度（2026-07-15）

- 已完成从现有 SSH 连接派生 Mosh 会话，复用主机指纹校验、密码/私钥/Agent 认证和 SSH 代理启动远端 `mosh-server`。
- 已完成应用内原生 PTY 托管、输入、输出、resize、关闭、日志、触发器、分屏、SFTP、监控和工作区恢复；一次性会话密钥仅通过 `MOSH_KEY` 子进程环境传递。
- 已内置由固定源码哈希构建的官方 Mosh 1.4.0 universal `mosh-client`，最低 macOS 13，仅链接系统库，并随包分发 GPL-3.0-or-later 与 protobuf BSD-3-Clause 许可证。
- 已完成 UDP 端口范围配置、远端缺失诊断、分块断线提示检测、NAT 公网目标修复、旧配置默认值和会话生命周期自动测试。
- 腾讯云 Ubuntu 24.04 x86_64 已完成远端握手、公网 UDP 会话建立、输入和正常退出短测；测试会话已清理。Wi-Fi/IP 切换、30 秒断网恢复、自动化屏幕 resize 捕获和 Intel 真机运行仍待对应环境验收，不虚报为已通过。

### 范围

- 新增 Mosh 连接模式，可从现有 SSH 连接派生，不复制密码和主机资料。
- 通过 SSH 启动远端 `mosh-server`，解析端口和密钥后启动受管本地客户端。
- 支持 UDP 端口范围、断网等待、IP 切换、恢复状态和代理限制说明。
- 共用现有标签、终端偏好、日志、触发器和关闭流程。
- 缺少依赖时提供应用内安装说明，不静默回退成不同协议。

### 交付策略

- 首版优先使用受管 `mosh-client` sidecar，并评估许可证、体积和 universal 构建。
- 若不适合内置，则提供应用内引导安装和依赖诊断，不要求用户手工拼接命令。

### 验收标准

- Wi-Fi 切换、客户端 IP 改变和 30 秒断网后会话可恢复。
- UDP 端口阻断时 10 秒内给出明确诊断。
- 会话密钥不进入日志、数据库或进程列表。
- Apple Silicon 与 Intel 均通过启动和关闭验证。

## 6. 阶段 3：高级 SSH 与密钥认证

### 当前进度（2026-07-15）

- SSH Certificate 已完成：连接模型与数据库迁移、私钥/证书双安全 Bookmark、OpenSSH 用户证书解析、CA/Key ID/序列号/主体/有效期展示、过期与未生效阻断、备份/复制/清理以及 `libssh2` 证书认证。
- 本机真实 OpenSSH `sshd` 已使用临时 Ed25519 CA、`TrustedUserCAKeys` 和短期用户证书完成认证及命令执行；测试 CA、密钥、证书和服务进程均由夹具退出时清理。
- X11 核心实现已完成：按连接默认关闭的真实 `x11-req`、入站 channel 接收、本机 Unix/loopback display 限制、一次性假 cookie 校验与真实 cookie 替换、64 KB 首包上限、8 channel 限制、关闭及重连清理。真实 OpenSSH 已验证远端 `DISPLAY` 建立；本机缺少 XQuartz，因此图形窗口端到端验收仍待外部环境。
- FIDO2 核心实现已完成：新增独立 `fido2Agent` 认证方式，从 OpenSSH Agent 的长度前缀公钥 blob 中只筛选 `sk-ssh-ed25519`、`sk-ecdsa-sha2-nistp256` 及对应用户证书，普通 RSA/Ed25519 Agent 身份不会被误选；编辑器显示算法、备注与 OpenSSH SHA-256 指纹，并提供无身份、触摸/PIN、取消和设备拔出/签名失败的可操作诊断。当前没有实体安全密钥，真机认证仍待外部验收。
- Touch ID Vault 核心实现已完成：同步口令可选写入 Data Protection Keychain，使用 `BIOMETRY_CURRENT_SET` 与 `AccessibleWhenPasscodeSetThisDeviceOnly` 绑定当前 Mac 和当前指纹集合；账户只保存规范化同步目录的 SHA-256，不保存原路径。读取口令和同步操作完全在 Rust 后端完成，口令不返回前端，内存副本在释放时清零；手动口令始终保留为恢复路径。真实 Touch ID 弹窗的保存、解锁、取消和指纹变化仍待应用内人工验收。

### X11 转发

- 按连接启用，默认关闭，仅允许可信主机。
- 探测 XQuartz、`DISPLAY` 和本地 socket，显示依赖诊断。
- SSH 层实现真实 X11 request、cookie 隔离和 channel 转发，不使用外部 shell 拼接。
- 支持会话关闭清理和 channel 数量限制。

### SSH Certificate

- 导入 OpenSSH 用户证书，展示 CA、有效期、主体和关键选项。
- 连接编辑器成对选择私钥与证书，过期前提示。
- 不在 CNshell 内保存 CA 私钥，也不首期实现证书签发服务。

### FIDO2 与 Touch ID

- 支持 `sk-ssh-ed25519` 和 `sk-ecdsa-sha2-nistp256` 硬件密钥认证。
- 区分触摸、PIN、驻留密钥、取消和设备拔出错误。
- Touch ID 首期用于解锁本地 Vault/同步密钥，不伪装为远端 SSH 生物认证。
- 生物识别失败时提供安全恢复路径。

### 验收标准

- XQuartz 上真实启动无特权 X11 应用并正确清理。
- OpenSSH 证书有效、过期和主体不匹配场景均有测试。
- 至少一种真实 FIDO2 设备完成认证、取消和拔出失败验证。

## 7. 阶段 4：RDP 深度集成

### 当前进度（2026-07-15）

- 已完成技术路线评估并选择独立 SDL 窗口深度联动，评估记录见 `docs/RDP_TECHNICAL_EVALUATION.md`；不把普通独立窗口冒充为画面内嵌。
- 已完成受管 FreeRDP 状态标记、真实 `connecting/online/reconnecting/closed/failed` 生命周期、手动关窗退出 131 分类、窗口启动位置跟随 CNshell、macOS 聚焦/隐藏、全屏与 FreeRDP 显示器选择。
- 已完成连接级显示与性能设置：动态分辨率/适应窗口/原始尺寸、自动/低带宽/均衡/高画质、文本剪贴板、远端声音、麦克风确认和单目录读写 Bookmark；参数仅通过 stdin，sidecar 继续崩溃隔离。
- 已完成参数、显示器解析、状态标记、窗口关闭、目录安全 Bookmark、前端设置和状态按钮测试；universal FreeRDP 修订版 4 已重建，真实 Windows 画面/IME/剪贴板/重连仍待外部主机。

### 技术验证门槛

现有 SDL FreeRDP 使用独立窗口。内嵌前先比较三条路线：

1. FreeRDP 输出到共享内存或 Metal 纹理，由 CNshell 转发输入。
2. 使用原生 NSView/Metal 子视图嵌入 Tauri 主窗口。
3. 保留独立窗口，但实现标签与窗口的定位、聚焦、最小化和全屏联动。

技术验证必须评估帧率、延迟、IME、键盘抓取、剪贴板、多显示器、动态分辨率、可访问性、崩溃隔离和 FreeRDP 升级成本。若内嵌无法稳定达标，选择第三条深度联动方案，不牺牲稳定性强行内嵌。

### 功能范围

- 标签真实反映连接中、在线、重连、关闭和失败。
- 支持缩放、键鼠、剪贴板、动态分辨率和全屏。
- 增加显示器选择、画质/带宽、音频重定向和磁盘映射开关。
- 重定向默认最小权限并显示安全提示。
- 保留 sidecar 崩溃隔离、日志脱敏和退出清理。

### 验收标准

- Windows 10/11 和 Windows Server 至少各一台真实主机通过。
- 键盘、中文输入、鼠标、复制粘贴、缩放和分辨率变化无错位。
- 手动关闭、断网、注销和认证失败状态分类正确。
- RDP 异常退出不拖垮 CNshell 主进程。

## 8. 阶段 5：高级自动化

### 当前进度（2026-07-15）

- 已完成受限自动化的持久化定时任务第一批：一次执行、固定间隔和 Cron 表达式校验。
- 已完成单次调度幂等更新、错过执行策略（跳过/恢复后执行一次）、后台任务 ID、取消和“立即运行”入口。
- 定时任务只复用已有受限步骤，不执行任意 Python、插件代码或用户登录 shell；应用退出期间不会偷偷常驻执行。
- 已完成受控 Python 运行器：使用固定版本 RustPython 解析器，只接受 `cnshell.command`、`wait_for`、`require`、`upload` 和 `download` 白名单调用；脚本编译为受限步骤，不启动系统 Python。
- 已完成 manifest 权限、脚本 SHA-256、目标与步骤预览、高风险命令警告、精确本地路径授权、取消和后台任务入口。
- 已完成结构化操作录制：只接收命令面板事件，不监听 xterm 原始键盘；包含密码、令牌、私钥或 `sshpass` 等敏感模式的命令不进入录制；录制可转换为受限计划。

### Python 脚本

- 不直接使用系统 Python；采用固定版本、受控运行器或独立 sidecar。
- 默认无网络、无 Keychain、无任意本地文件和无终端输入权限。
- 通过 manifest 授权连接只读信息、执行命令、传输指定文件和读取步骤结果。
- 运行前展示目标主机、权限、脚本哈希和预计操作。

### 操作录制

- 只录制 CNshell 内可结构化动作，不录制密码、原始键盘监听或其他应用输入。
- 转换为现有受限自动化步骤，保存前允许编辑和预览。
- 不可结构化或包含秘密的操作标记为人工步骤。

### 定时任务

- 支持一次、每日、每周和 Cron 表达式预览。
- 首版仅实现 CNshell 运行时调度，后台常驻另行评审。
- 明确错过执行、睡眠唤醒、重复运行、超时和失败重试策略。

### 验收标准

- 未授权脚本无法访问网络、Keychain 或任意文件。
- 录制过程不产生密码和原始输入泄漏。
- 睡眠唤醒和时区变化后不会重复执行同一实例。

## 9. 阶段 6：内置云同步客户端

### 当前进度（2026-07-15）

- 已完成原生 WebDAV 配置管理，URL 与用户名保存于 SQLite 工作区状态，密码只保存到独立 Keychain 服务。
- 已完成 HTTPS 强制策略（仅 loopback 测试允许 HTTP）、禁止 URL 内嵌凭据、禁用重定向、请求超时和 50 MB 响应上限。
- 已完成加密包上传/下载、远端版本副本、ETag `If-Match`/`If-None-Match` 并发保护、后台任务、取消检查和错误分类。
- 已完成分阶段字节进度事件与网络请求即时取消；启动时自动导入必须显式开启并另存同步口令到 Keychain，缺少口令时启动不联网。
- 已完成本机假 WebDAV 服务的 PUT/GET、Basic Auth 与二进制密文互操作测试；真实 WebDAV 账号、配额满和多设备并发仍待对应环境验收。

### 首期范围

- 首期实现 WebDAV 原生客户端，保留现有本地目录同步。
- 服务器地址和用户名存数据库，密码存 Keychain。
- 上传继续使用现有 Argon2id + AES-256-GCM 加密包，服务端只见密文。
- 支持手动同步、启动同步、冲突副本、版本保留、进度和取消。
- 不首期实现 CNshell 自有账号系统。

### 后续适配器

- iCloud Drive 使用系统文件提供器目录，不模拟 Apple 账号登录。
- Git 支持用户已有仓库，不保存托管平台密码。
- S3 兼容存储只在真实需求出现后实现。

### 验收标准

- 抓包和服务端文件扫描看不到主机、用户名、路径和凭据明文。
- 错误口令、断网、5xx、配额满和并发冲突不覆盖本地有效数据。
- 两台设备同步同 ID 连接时保留可识别冲突副本。

- 已完成 IP、主机名、用户名、路径、令牌、密钥和常见密码字段的默认脱敏；请求显示实际发送文本、脱敏类别、Provider 和模型。
- 已完成命令生成、错误解释、日志总结三种提示边界；结果只展示/复制，不自动执行、不接受主机密钥、不修改连接安全设置。

### 首期范围

- 自然语言生成命令，展示目标主机、最终命令和风险后由用户手动执行。
- 只解释用户选中的错误输出，不默认上传整个终端缓冲区。
- 根据用户明确选择的日志生成摘要和只读诊断步骤。
- 支持 BYOK，API Key 存 Keychain；预留本地 OpenAI-compatible endpoint。

### 数据边界

- 每次请求前展示将发送的文本，可编辑和取消。
- 默认脱敏 IP、主机名、用户名、路径、令牌和私钥片段。
- 禁止自动执行命令、接受主机密钥、发送凭据或修改安全设置。
- 不建立默认遥测、训练上传或云端会话历史。

### 验收标准

- 网络请求只能发往用户选择的 provider/endpoint。
- 未配置模型时完全离线且不产生后台网络错误。
- AI 命令继续经过现有高风险检测和预览流程。

## 11. 阶段 8：插件体系与团队版

### 当前进度（2026-07-15）

- 已完成插件 manifest v1 解析和边界校验：反向域名 ID、WASM 相对入口、权限数量、域名白名单和 API 版本。
- 已完成权限拆分与默认授予/拒绝报告；UI、连接元数据和终端只读可作为低风险默认权限，网络、目录、终端输入和凭据代理默认拒绝。
- 已完成未签名/未受信任发布者的明确阻断报告；当前只检查 manifest，不执行插件代码，也不把自签名当作信任。
- 已完成本地插件登记、SHA-256 固定、禁用、移除、最多 256 条审计保留和原子 JSON 安全导出；登记与审计通过同一 SQLite 事务写入。权限声明变化会生成独立审计事件并继续阻断执行。
- 当前没有受信任发布者根和 WASM/sidecar 沙箱，因此所有登记记录的 `enabled` 与 `executable` 都保持 `false`，不存在绕过登记直接运行第三方代码的入口。

### 插件体系

- 定义版本化 SDK、manifest、兼容范围和签名格式。
- 权限拆分为 UI、网络域名、指定目录、终端只读、终端输入、连接元数据和凭据代理。
- 默认拒绝敏感权限；凭据只通过短期代理使用，插件不能读取明文。
- 插件在独立进程或 WebAssembly 沙箱运行，崩溃与主程序隔离。
- 支持禁用、卸载、权限变更提示和供应链撤销。

### 团队协作

- 团队工作区、成员、角色和最小权限 RBAC。
- 连接安全分享采用端到端加密，服务端不能读取凭据明文。
- 多人终端首版只读观看，再评估协作输入和控制权移交。
- 审计日志记录成员、时间、目标和操作类型，默认不记录完整输出和秘密。
- 提供组织导出、成员移除、密钥轮换和设备撤销。

### 验收标准

- 恶意或崩溃插件不能读取未授权文件、网络、终端或 Keychain。
- 权限升级必须重新确认，禁用或卸载后权限立即失效。
- 团队成员移除后无法解密后续数据，密钥轮换不破坏可访问历史。

## 12. 阶段 9：遗留协议扩展

这些协议只在有真实设备和需求时开发，不阻塞主路线：

| 协议 | 计划定位 | 安全与实现要求 |
| --- | --- | --- |
| 本地 Shell | 可优先于其他遗留协议 | 已完成本机 PTY 会话、输入、尺寸调整、关闭、输出事件和独立日志；与 SSH 标签明确区分 |
| Serial | 网络设备和嵌入式设备 | 串口枚举、波特率/校验位/流控、独占访问和拔出恢复 |
| Telnet | 明确警告的兼容模式 | 已完成 TCP 会话、IAC 协商过滤、关闭/断线状态和未加密警告；不保存密码，避免公网误用 |
| X/Ymodem | 串口传输补充 | 复用 Serial 和传输队列，完成校验、取消和清理 |
| Kermit | 最低优先级 | 仅在真实设备需求出现后评估许可证和成本 |

## 13. 统一工程与安全要求

- 所有 IPC 参数有长度、数量、路径和枚举边界，Rust 为类型单一来源。
- 所有长任务立即返回任务 ID，支持查询、事件进度、取消和超时。
- 所有协议 session 有唯一 ID、明确状态机和幂等关闭流程。
- 所有凭据存 Keychain，数据库只保存引用和非秘密元数据。
- 二进制协议、AI、插件和同步不得阻塞 PTY reader 或 UI 渲染。
- 数据库迁移必须覆盖历史 schema 升级和失败回滚。
- 新设置必须有安全默认值，并明确是否进入备份、同步和诊断。
- 每阶段同步更新 `CHANGELOG.md`、用户手册、安全说明和验收记录。
