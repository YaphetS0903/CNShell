> 文档状态：v2.6 执行中（阶段 1–9 本机可实现代码主线和短时自动化验收已完成，生产 relay 邮箱验证、代理限速与监控部署代码已补齐，正式 updater 验签、回滚兼容和发布供应链固定已完成；正式发布、生产部署与外部真机验收待补）
>
> 制定日期：2026-07-14
>
> 适用范围：当前候选版之后尚未开发、未完整开发或尚未完成正式验收的能力

## 1. 目标与原则

本规划覆盖 Zmodem、Mosh、X11、RDP 内嵌、高级密钥、扩展自动化、AI、插件与团队协作、遗留协议、内置云同步客户端以及正式发布能力。已有 SSH、SFTP、监控、端口转发、受限自动化和加密目录同步继续作为稳定基线。

实施原则：

1. 每个阶段形成可独立使用、测试和回滚的交付物，不同时铺开全部功能。
2. 密码、私钥、终端内容和连接资料默认只留在本机；任何上传必须由用户明确选择范围。
3. 协议能力必须真实实现并完成互操作验收，检测到依赖不等于功能完成。
4. 高风险操作必须预览、确认、可取消并记录本机审计事件。
5. 第三方组件必须固定版本和哈希、审计许可证、构建 universal binary 并纳入签名。
6. 长测、1 GB 文件和跨设备验收只在阶段完成时执行，不纳入每次普通开发门禁。

## 4. 阶段 1：Zmodem 完整传输

### 当前进度（2026-07-15）

- 已完成 SSH 原始字节流握手检测、CRC 校验、协议帧接管和日志隔离。
- 已完成 macOS 原生上传/下载授权、多文件、空文件、中文空格文件名、自动冲突改名、原子下载、取消、授权超时、断线恢复和实时进度 UI。
- 已与腾讯云 Linux x86_64 `lrzsz` 完成真实双向小文件及多文件互操作并校验远端退出状态。
- 已通过 Rust 全量测试、前端全量测试和 arm64 应用打包安装；100 MB、磁盘满和人工中途取消作为阶段验收专项，不在普通迭代中重复执行。

### 范围

- 在 SSH 原始字节流中识别 `rz/sz` 握手序列，不把协议字节写入 xterm。
- 建立每会话状态机：空闲、协商、上传、下载、取消、成功和失败。
- 上传和下载都通过 macOS 原生文件选择器授权。
- 支持单文件、多文件、进度、速度、取消、超时、重试和冲突策略。
- 传输时暂停普通终端输入，结束后恢复同一 PTY。
- 对文件名、路径、大小和协议帧设置上限，拒绝远端任意覆盖本地路径。

### 技术方案

- Rust 侧接管 SSH channel 原始输入/输出并运行 Zmodem 编解码器。
- 前端仅接收传输状态事件，不解析二进制协议。
- 优先评估许可证兼容的 Rust 实现；若引入 sidecar，固定源码哈希并分发许可。
- 复用现有传输队列，但与 SFTP/SCP 任务明确区分协议类型。

### 验收标准

- 与 Linux `lrzsz` 完成上传、下载、多文件、空文件和中文空格文件名互操作。
- 100 MB 文件 SHA-256 一致；取消不暴露为完整文件。
- 握手误识别不会吞掉普通终端输出。
- 断线、超时、磁盘满和远端拒绝均能恢复终端并给出可操作错误。

### 验收标准

- Wi-Fi 切换、客户端 IP 改变和 30 秒断网后会话可恢复。
- UDP 端口阻断时 10 秒内给出明确诊断。
- 会话密钥不进入日志、数据库或进程列表。
- Apple Silicon 与 Intel 均通过启动和关闭验证。

## 6. 阶段 3：高级 SSH 与密钥认证

### 当前进度（2026-07-15）

- SSH Certificate 已完成：连接模型与数据库迁移、私钥/证书双安全 Bookmark、OpenSSH 用户证书解析、CA/Key ID/序列号/主体/有效期展示、过期与未生效阻断、备份/复制/清理以及 `libssh2` 证书认证。
- 证书连接前会按 OpenSSH 语义校验主体：非空 principals 必须精确、区分大小写地包含目标用户名；空 principals 保留不限制用户名的语义，主体不匹配在发网前阻断。
- 本机真实 OpenSSH `sshd` 已使用临时 Ed25519 CA、`TrustedUserCAKeys` 和短期用户证书完成认证及命令执行；测试 CA、密钥、证书和服务进程均由夹具退出时清理。
- X11 核心实现已完成：按连接默认关闭的真实 `x11-req`、入站 channel 接收、本机 Unix/loopback display 限制、一次性假 cookie 校验与真实 cookie 替换、64 KB 首包上限、8 channel 限制、关闭及重连清理。真实 OpenSSH 已验证远端 `DISPLAY` 建立；本机缺少 XQuartz，因此图形窗口端到端验收仍待外部环境。
- FIDO2 核心实现已完成：新增独立 `fido2Agent` 认证方式，从 OpenSSH Agent 的长度前缀公钥 blob 中只筛选 `sk-ssh-ed25519`、`sk-ecdsa-sha2-nistp256` 及对应用户证书，普通 RSA/Ed25519 Agent 身份不会被误选；编辑器显示算法、备注与 OpenSSH SHA-256 指纹，并提供无身份、触摸/PIN、取消和设备拔出/签名失败的可操作诊断。当前没有实体安全密钥，真机认证仍待外部验收。
- Touch ID Vault 核心实现已完成：同步口令可选写入 Data Protection Keychain，使用 `BIOMETRY_CURRENT_SET` 与 `AccessibleWhenPasscodeSetThisDeviceOnly` 绑定当前 Mac 和当前指纹集合；账户只保存规范化同步目录的 SHA-256，不保存原路径。读取口令和同步操作完全在 Rust 后端完成，口令不返回前端，内存副本在释放时清零；手动口令始终保留为恢复路径。真实 Touch ID 弹窗的保存、解锁、取消和指纹变化仍待应用内人工验收。

### X11 转发

- 按连接启用，默认关闭，仅允许可信主机。
- 探测 XQuartz、`DISPLAY` 和本地 socket，显示依赖诊断。
- SSH 层实现真实 X11 request、cookie 隔离和 channel 转发，不使用外部 shell 拼接。
- 支持会话关闭清理和 channel 数量限制。

### SSH Certificate

- 导入 OpenSSH 用户证书，展示 CA、有效期、主体和关键选项。
- 连接编辑器成对选择私钥与证书，过期前提示。
- 不在 CNshell 内保存 CA 私钥，也不首期实现证书签发服务。

### FIDO2 与 Touch ID

- 支持 `sk-ssh-ed25519` 和 `sk-ecdsa-sha2-nistp256` 硬件密钥认证。
- 区分触摸、PIN、驻留密钥、取消和设备拔出错误。
- Touch ID 首期用于解锁本地 Vault/同步密钥，不伪装为远端 SSH 生物认证。
- 生物识别失败时提供安全恢复路径。

### 验收标准

- XQuartz 上真实启动无特权 X11 应用并正确清理。
- OpenSSH 证书有效、过期和主体不匹配场景均有测试。
- 至少一种真实 FIDO2 设备完成认证、取消和拔出失败验证。

## 7. 阶段 4：RDP 深度集成

### 当前进度（2026-07-15）

- 已完成技术路线评估并选择独立 SDL 窗口深度联动，评估记录见 `docs/RDP_TECHNICAL_EVALUATION.md`；不把普通独立窗口冒充为画面内嵌。
- 已完成受管 FreeRDP 状态标记、真实 `connecting/online/reconnecting/closed/failed` 生命周期、手动关窗退出 131 分类、窗口启动位置跟随 CNshell、macOS 聚焦/隐藏、全屏与 FreeRDP 显示器选择。
- 已完成连接级显示与性能设置：动态分辨率/适应窗口/原始尺寸、自动/低带宽/均衡/高画质、文本剪贴板、远端声音、麦克风确认和单目录读写 Bookmark；参数仅通过 stdin，sidecar 继续崩溃隔离。
- 已完成参数、显示器解析、状态标记、窗口关闭、目录安全 Bookmark、前端设置和状态按钮测试；universal FreeRDP 构建脚本修订版 5 已关闭可选宿主 JSON 库探测，避免 x86_64 构建误链接 arm64 Homebrew 依赖；真实 Windows 画面/IME/剪贴板/重连仍待外部主机。

### 技术验证门槛

现有 SDL FreeRDP 使用独立窗口。内嵌前先比较三条路线：

1. FreeRDP 输出到共享内存或 Metal 纹理，由 CNshell 转发输入。
2. 使用原生 NSView/Metal 子视图嵌入 Tauri 主窗口。
3. 保留独立窗口，但实现标签与窗口的定位、聚焦、最小化和全屏联动。

技术验证必须评估帧率、延迟、IME、键盘抓取、剪贴板、多显示器、动态分辨率、可访问性、崩溃隔离和 FreeRDP 升级成本。若内嵌无法稳定达标，选择第三条深度联动方案，不牺牲稳定性强行内嵌。

### 功能范围

- 标签真实反映连接中、在线、重连、关闭和失败。
- 支持缩放、键鼠、剪贴板、动态分辨率和全屏。
- 增加显示器选择、画质/带宽、音频重定向和磁盘映射开关。
- 重定向默认最小权限并显示安全提示。
- 保留 sidecar 崩溃隔离、日志脱敏和退出清理。

### 验收标准

- Windows 10/11 和 Windows Server 至少各一台真实主机通过。
- 键盘、中文输入、鼠标、复制粘贴、缩放和分辨率变化无错位。
- 手动关闭、断网、注销和认证失败状态分类正确。
- RDP 异常退出不拖垮 CNshell 主进程。

这些协议只在有真实设备和需求时开发，不阻塞主路线：

| 协议 | 计划定位 | 安全与实现要求 |
| --- | --- | --- |
| 本地 Shell | 可优先于其他遗留协议 | 已完成本机 PTY 会话、输入、尺寸调整、关闭、输出事件和独立日志；与 SSH 标签明确区分 |
| Serial | 网络设备和嵌入式设备 | 已完成串口枚举、波特率/数据位/校验位/停止位/流控、DTR/RTS、独占访问、输入输出生命周期和同路径拔出重连；实体串口设备互操作待补 |
| Telnet | 明确警告的兼容模式 | 已完成 TCP 会话、IAC 协商过滤、关闭/断线状态和未加密警告；不保存密码，避免公网误用 |
| X/Ymodem | 串口传输补充 | 已完成 Xmodem 128/1K、Checksum/CRC、Ymodem Batch、进度、取消、原子落盘和双端回环；真实设备与外部实现互操作待补 |
| Kermit | 最低优先级 | 已内置受管 G-Kermit 2.01 universal sidecar，完成管道互操作、取消与安全落盘；实体串口及第三方设备互操作待补 |

## 13. 统一工程与安全要求

- 所有 IPC 参数有长度、数量、路径和枚举边界，Rust 为类型单一来源。
- 所有长任务立即返回任务 ID，支持查询、事件进度、取消和超时。
- 所有协议 session 有唯一 ID、明确状态机和幂等关闭流程。
- 所有凭据存 Keychain，数据库只保存引用和非秘密元数据。
- 二进制协议、AI、插件和同步不得阻塞 PTY reader 或 UI 渲染。
- 数据库迁移必须覆盖历史 schema 升级和失败回滚。
- 新设置必须有安全默认值，并明确是否进入备份、同步和诊断。
- 每阶段同步更新 `CHANGELOG.md`、用户手册、安全说明和验收记录。

## 15. 下一步开发顺序

阶段 0 中需要 Apple 证书、更新服务和跨设备资源的工作作为并行发布轨道推进，不阻塞本机协议功能开发；但 AI、插件和团队协作进入公开交付前必须完成阶段 0。

阶段 1–9 的本机可实现代码主线、短时自动化门禁以及多人终端在线客户端已经完成。RDP 当前采用“独立 FreeRDP 窗口深度联动”路线，未把不稳定的像素搬运强行伪装成主工作区内嵌。下一步转入 **正式发布、生产服务与外部验收轨道**：

外部环境到位后先运行 `npm run preflight:external`；该只读预检和脱敏证据格式见 `docs/EXTERNAL_ACCEPTANCE.md`。`READY` 仅表示前置条件存在，不等同对应真机场景已经通过。

1. 有 XQuartz 环境时补做 X11 图形窗口端到端验收；有实体安全密钥时补做 FIDO2 认证、取消和拔出；在当前 Mac 上人工验证 Touch ID 保存、解锁、取消和指纹变化。这些外部验收不阻塞后续代码开发，但不得虚报为通过。
2. 发布者根、签名安装、撤销、摘要失效、无 WASI WASM 沙箱、连接元数据、终端选区、受限日志、网络域名、用户授权目录、受控终端输入和一次性 `connectionTest` 凭据代理已完成。后续只在出现真实需求时扩展 ABI，不能用通用 WASI 绕过权限模型。
3. 团队工作区、本地 RBAC、组织目录导出、设备密钥、离线 E2E 分享、客户端 relay 账号/工作区同步、生产邮箱验证、房间 REST/WebSocket、端到端签名加密帧、控制租约、防重放、有界断线补帧、观看/控制 UI，以及 TLS/WSS 代理、限速、Prometheus/Alertmanager 配置、优雅停机和默认强制 `age` 的备份/恢复运维代码已完成；官方 v1.3.1 release 的 Sigsum 验证和本机功能演练也已通过。本机/CI 证据不替代生产验收；下一项必须把模板部署到正式域名，验证真实 SMTP 与告警投递，并用生产 identity 执行加密异地备份恢复；随后仍需至少两台真实设备，才能验收跨设备观看、控制移交与真实网络断线恢复。
4. Serial 已接入设备枚举、独占打开、参数配置、输入生命周期和拔出重连；X/Ymodem 已完成内置协议、应用内授权、取消、进度和回环测试；Kermit 已完成官方 helper 的 universal 构建、应用内受管桥接和管道互操作。在有实体设备时补全部串口协议外部互操作，不把本机管道测试写成硬件通过。

RDP 的共享内存/Metal、原生 NSView/Metal 子视图研究记录已归档到 `docs/RDP_TECHNICAL_EVALUATION.md`；在有 Windows 真机和可量化收益前不重构现有稳定 RDP 路径。
