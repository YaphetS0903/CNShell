# CNshell 用户手册

## 首次连接

1. 点击工具栏 `+`，选择 SSH，填写名称、主机、端口和用户名。
2. 选择密码、私钥、SSH Certificate、SSH Agent 或 FIDO2 硬件密钥。私钥可用“选择…”打开系统文件选择器，也可输入本机绝对路径；密码和私钥口令写入 macOS Keychain 或 Windows 凭据管理器，私钥内容不会复制进 CNshell。macOS 以只读 security-scoped Bookmark 保存路径授权；Windows 使用规范化绝对路径，并在凭据项容量允许时保存冗余路径记录，超长路径仍保留在连接资料中且每次访问都会重新校验。FIDO2 只尝试 Agent 中的 `sk-*` 身份，普通 Agent 密钥不会被误用。
3. 首次连接时在服务器控制台或向管理员核对 SHA-256 主机指纹，再点击信任。
4. 指纹后续发生变化时 CNshell 会阻止连接；确认服务器确实重装或更换密钥后再更新记录。

## 终端与文件

- 终端标签互相独立；macOS 使用 `⌘1…9`、`⌘F`、`⌘K` 和 `⌘+` / `⌘-` / `⌘0`，Windows 使用对应的 `Ctrl` 组合键来切换标签、搜索、清屏和调整字号。
- “设置 → 终端”可调整字体、字号、行高、回滚行数、光标形状/闪烁和终端配色；保存后所有已打开终端即时生效，不会重建 SSH 会话。
- 连接编辑页可启用“覆盖全局终端偏好”，只让该连接使用独立外观和缓冲设置。
- 文件面板双击目录进入、双击小于 10 MB 的 UTF-8 文件编辑。
- 左侧目录树可逐级展开或折叠；点击目录名称会在右侧打开，并自动展开当前路径的祖先目录。
- 文件行右键可下载、上传到所选目录、新建文件/文件夹、打开、编辑、复制路径、重命名、改权限、压缩/解压和删除；新建文件不会覆盖已有同名文件。
- 保存文本会检查远端修改时间，先写临时文件再原子替换。
- 下载先写 `.cnshell-part`；成功同步后才改成目标文件名。
- 传输队列支持暂停、恢复、取消和失败重试。
- “上传文件夹”与选中目录后的“下载文件夹”会自动打包传输；覆盖时先完成新目录，再通过同级隐藏备份交换，失败或取消会清理临时归档并保留原目标。

## 监控

CNshell 每 2 秒通过独立 SSH Exec Channel 读取 `/proc`、`df`、`ip` 和 `ps`。CPU、网络上下行与延迟保留最近 5 分钟趋势，同时提供当前数值作为图表替代。监控失败不会中断终端与 SFTP，可在设置中调整刷新频率或暂停。

## 代理与隧道

在设置中创建 SOCKS5、HTTP CONNECT 或 SSH 跳板代理，再在连接编辑页选择。连接行的更多菜单可创建本地、远程和动态 SOCKS5 转发。非 loopback 监听可能暴露端口，必须自行配置防火墙。

连接可拖放到嵌套文件夹，也可从连接行的更多菜单选择“移动到文件夹”；删除的连接进入“已删除项目”，可恢复或永久删除。

主机密钥策略默认要求首次人工核对 SHA-256 指纹。可选的“自动信任首次密钥”仅适合可信网络：它会自动记录第一次看到的算法与指纹，之后任何变化仍会阻止连接。

## RDP

直接创建 RDP 连接即可。CNshell 应用包已内置当前平台和架构对应的 `sdl-freerdp`：macOS 提供 universal helper，Windows 分别提供 x64/ARM64 helper；用户不需要通过 Homebrew 或终端另行安装 FreeRDP。受管 SDL 窗口支持动态分辨率、文本剪贴板和自动重连。连接编辑页可选择窗口/全屏、显示器、缩放、画质/带宽和远端声音；麦克风与本地目录映射默认关闭，启用前会确认权限。密码写入 macOS Keychain 或 Windows 凭据管理器；目录映射在 macOS 使用读写 Bookmark，在 Windows 使用用户选择的本机绝对路径。参数和密码仅通过 sidecar stdin 传递，不进入命令行参数或环境变量。CNshell 标签会显示 connecting/online/reconnecting/closed/failed 状态，可切到或隐藏受管远程窗口；关闭标签会终止对应远程桌面进程。

## 本地 Shell 与 Telnet

创建连接时可选择“本地 Shell”。macOS 使用当前登录 Shell 打开独立 PTY；Windows 通过 ConPTY 按 `pwsh.exe`、`powershell.exe`、`cmd.exe` 的顺序选择本地 Shell。它支持标签、分屏、终端偏好和会话日志，但不会启用远程监控或 SFTP。

Telnet 仅用于受控内网中的遗留设备。连接和标签会持续标记“未加密”，CNshell 不提供 Telnet 密码字段，也拒绝从导入或 IPC 保存 Telnet 凭据。需要登录时只能在已建立的明文终端中手动输入；公网或不可信网络应使用 SSH。

Serial 连接会枚举 macOS 可见的 `/dev/cu.*` 设备或 Windows 的 `COM1` 至 `COM256`；macOS 也可手动输入其他 `/dev` 路径，Windows 的 `COM10+` 会自动使用系统要求的扩展设备路径。选择设备后可设置波特率、数据位、校验位、停止位、软件/硬件流控及 DTR/RTS；设备默认独占打开。设备拔出时会话进入“重连中”，只尝试恢复原设备路径，重新插入后可继续使用；Serial 不提供远程监控或 SFTP。

Serial 会话的“文件”面板提供 Xmodem 128/1K CRC、传统 Checksum、Ymodem Batch 和 Kermit Batch 上传/下载。所有本地路径必须通过系统文件选择器授权；传输期间普通终端输入暂停，完成、取消或失败后恢复。Ymodem/Kermit 下载使用远端文件名和真实长度，同名文件自动改名；Xmodem 协议不包含文件名与真实长度，因此每次只传一个文件，并保留最后一个 128/1024 字节块中的 `0x1A` 填充。Kermit 使用应用内置的当前平台 helper：macOS 为 universal，Windows 为对应的 x64/ARM64 版本，不需要另外安装命令行工具。

## 备份与诊断

- “安全导出”不含任何凭据。
- “加密导出凭据”要求至少 8 位口令，使用 Argon2id + AES-256-GCM；口令无法找回。
- 诊断 JSON 不包含主机、用户名、命令或文件路径，可在提交问题前自行检查。
- “设置 → 反馈与诊断”可打开预填 CNshell 版本、操作系统版本和架构的错误报告，也可提交功能建议或查看当前 Release。
- 诊断文件导出成功后可在 macOS Finder 或 Windows 文件资源管理器中显示；CNshell 不会自动上传诊断文件。

## MCP 与 AI 客户端

CNshell 内置本机 MCP Server，可让 Codex、Claude Code 或其他支持 stdio MCP 的客户端使用已保存的 SSH 连接。MCP 默认关闭，不需要另装命令行组件。

1. 打开“设置 → MCP 服务”，启用 MCP。Broker 只监听 `127.0.0.1` 随机端口。
2. 创建一个客户端，例如 `Codex`，然后只勾选它可以使用的 SSH 连接、工具和远端授权根。
3. 保存授权后复制 Codex TOML 或通用 JSON，放入对应 MCP Host 的配置中。配置只含内置 sidecar 路径、客户端 ID 和 discovery 路径，不含密码、私钥或 Broker secret。
4. 保持 CNshell 运行。读取连接、目录、文本和系统信息会按已授权范围执行；命令、远端写入、删除、上传和下载会在右侧 MCP 审批抽屉显示完整目标与预览，120 秒内选择“允许一次”或“拒绝”。只有 `pwd`、`whoami`、`id`、`uptime`、`date`、指定的 `uname` 读取命令，以及不含格式化/展开的单字面量 `printf` 可保存为精确规则；规则只保存完整命令文本的 SHA-256 摘要，不保存历史命令明文。
5. 本地上传必须先在设置中选择一个只读文件或目录；下载必须先选择可写目标目录。客户端只能传授权 ID 和相对路径，不能指定任意本机绝对路径。当前界面创建的是 24 小时内有效的一次性授权，成功批准后即被消费。
6. 审计页只显示请求元数据，可使用“导出审计”写入脱敏 JSON；导出不包含命令输出、文件正文、本地绝对路径或凭据。
7. 不再使用时关闭 MCP 或撤销客户端。撤销会取消该客户端正在执行的请求，并使会话、待审批、本地路径授权和系统凭据立即失效。
8. 已保存的精确命令规则可在对应客户端设置中查看和逐条撤销。页面只显示连接、工具、SHA-256 摘要、创建时间和最近使用时间；每个客户端最多 256 条，撤销后下一次相同命令会重新请求审批。

首版提供 13 个工具：连接清单、短期会话开关、远端目录、文本读取、系统信息、非交互命令、文本写入、建目录、重命名、删除、上传和下载。MCP 不提供交互式 PTY、`sudo` 密码输入、RDP、Mosh、Serial、Keychain/凭据管理器或任意本地路径访问。

支持 Resources 的 Host 可读取 4 项内容：`cnshell://guide/mcp` 与 `cnshell://guide/security` 是内置静态说明；`cnshell://connections` 只列出当前客户端被明确授予连接清单工具权限的连接，主机和用户名仍受隐私开关控制；`cnshell://audit/recent` 只返回当前客户端最近 100 条脱敏审计元数据。动态 Resources 会经过 Broker 身份、授权、限流和审计校验，不会返回连接备注、凭据、request ID、目标摘要、命令、路径、正文或输出。支持 Prompts 的 Host 还可使用连接诊断和命令安全评审模板；Prompt 不会自动执行命令。

## 高级效率与协议

- 终端工具栏可打开智能命令栏、会话日志、高亮/通知、多主机批量执行、Copy Mode、跨标签搜索和逐行时间戳。
- 设置中的“高级协议与转发”会探测 Zmodem、SCP、Mosh、X Server 与 SSH Agent。Agent 转发必须逐连接启用，仅用于完全可信主机；设置只影响下一次连接。
- SFTP 子系统不可用时，普通文件传输会在同一条已认证、已校验指纹的 SSH 连接上自动尝试 SCP。SCP 上传只接受用户明确选择的覆盖策略。
- “受限任务编排”按顺序执行命令、匹配/条件和文件传输步骤。运行前必须检查最终预览；任一步失败即停止，可取消或修正后重试。
- “定时任务”支持一次执行、固定间隔、每日、每周或 Cron 计划，并显式保存 IANA 时区；错过执行可选择跳过或恢复后执行一次。调度器先保存运行游标再启动受限步骤，DST 回拨不会把同一墙上时间执行两次，CNshell 退出时不会在后台常驻。
- “受限 Python”只接受 `cnshell.command`、`wait_for`、`require`、`upload` 和 `download` 调用。每个脚本必须声明目标、权限和本地文件授权，运行前显示 SHA-256 与步骤预览；不支持 `import`、系统 Python、任意文件/网络 API 或子进程。
- “操作录制”只记录命令面板提交的结构化命令，不监听终端原始键盘。密码、令牌、私钥和 `sshpass` 等敏感命令会被排除，停止录制后可编辑并转成受限计划。
- “本地目录加密同步”只操作用户选择的目录，可指向 macOS 的 iCloud Drive、Windows 的 OneDrive、WebDAV 挂载点或 Git 检出目录。口令默认不保存；可明确选择用 Touch ID 或 Windows Hello 保存为这台设备专属的受保护凭据。
- “WebDAV 同步”直接连接 HTTPS WebDAV 地址，用户名保存在本地数据库，密码保存在 macOS Keychain 或 Windows 凭据管理器。上传前保留远端版本并使用 ETag 条件写入；另一设备已修改文件时会拒绝覆盖。下载按字节显示进度并可在响应途中取消，配额不足、服务失败和响应过大都有独立错误。同步内容始终是加密包，凭据同步默认关闭。
- 启动自动导入默认关闭。开启后需另存同步口令到系统凭据库，CNshell 下次启动只下载并导入远端加密包；未保存口令时不会在启动阶段联网。
- “AI 辅助”先保存用户选择的 Provider 和模型，再对选中文本生成脱敏预览。只有点击确认后才发送，API Key 保存在系统凭据库；响应可取消且限制为 64 KB，命令结果需要用户手动复制和执行，错误解释与日志总结不会自动修改主机。
- “插件信任与沙箱”可以导入 Ed25519 发布者根、检查 manifest、登记、启用、运行、禁用、撤销和导出审计。导入前应通过独立渠道核对发布者公钥指纹；撤销会立即禁用该发布者全部插件。CNshell 同时固定 manifest 与 WASM SHA-256，并在每次运行前重新校验。沙箱无 WASI、通用网络/文件、系统凭据库或原始终端访问，只按 manifest 和用户本次授权开放精确域名 HTTPS GET、授权目录只读、当前终端选区、需再次确认的单次终端输入和 `connectionTest` 凭据代理；插件只接受导出 `cnshell_main: () -> i32` 的受信任 WASM。详见 `docs/PLUGIN_SDK.md`。移除登记不会删除原 manifest 或 WASM 文件。
- “团队工作区与 RBAC”可建立 Owner/Admin/Operator/Viewer 成员，查看当前权限、密钥 epoch 与最近审计。只有 Owner 能管理 Owner；Admin 可管理普通成员；Operator 可使用连接和控制终端；Viewer 只读。Owner/Admin 可导出只含工作区、成员、设备公钥/指纹和审计元数据的组织目录；导出不含连接、凭据、token 或终端正文。配置 HTTPS relay 并登录后，可发布工作区、创建或接受在线邀请，并以服务端成员/设备目录为准同步。
- 首次分享前在团队工作区创建本机设备身份，私钥会保存到 macOS Keychain 或 Windows 凭据管理器。通过 `.cnshelldevice` 交换公钥并核对指纹后，可选择一至 64 台活动设备导出 `.cnshellshare`。包含密码时会再次确认；私钥/证书、代理、备注、启动命令和环境变量不会进入分享。接收端打开文件后先查看主机、协议、epoch 和是否含凭据，再明确导入；预览 5 分钟后失效。工作区轮换 epoch 后，历史分享只对当时已列为接收者且当前仍活动的设备继续可读。
- 终端标签栏的“在线团队终端”可从当前在线 SSH 会话建立主持房间，选择远端设备发送邀请，并授予 30 秒至 5 分钟控制权。参与者在“加入”页接受邀请后默认只读观看；只有本机设备持有未过期租约时终端才接受输入。关闭面板不会断开房间，使用“关闭房间”才会离开或结束会话。当前功能面向 loopback 或自行部署的测试 relay，正式公网服务和跨设备真机验收尚未完成。
