# CNshell 竞品功能审计与后续路线图

> 调研日期：2026-07-12
> 调研范围：FinalShell、Termius、SecureCRT、Tabby、iTerm2、Warp 公开功能资料
> 目标：在保持本地优先、安全和 macOS 工作效率的前提下，确定 CNshell 下一阶段的功能顺序。

> 实施核验：2026-07-12 已完成本机可实施项。状态与证据见第 7 节；依赖外部协议程序、X Server、发行基础设施或尚未稳定的权限模型之项目不会虚报为完成。

## 1. 当前基线

CNshell 当前已经具备以下核心能力：

- SSH 密码、私钥和 SSH Agent 认证，严格主机指纹校验。
- SOCKS5、HTTP CONNECT、SSH Jump 及本地、远程、动态端口转发。
- 多标签终端、双窗格拆分、搜索、快捷命令、历史和工作区恢复。
- SFTP 目录树、文件操作、文本编辑、拖放及文件/文件夹后台传输。
- CPU、内存、Swap、进程、网络、延迟和磁盘实时监控。
- 系统信息、连接诊断、加密备份、软删除和脱敏诊断导出。
- macOS Keychain、浅色/深色/高对比主题及 universal App/DMG。
- 随应用分发的 universal FreeRDP SDL sidecar 适配层。

目前基础 SSH/SFTP/监控工作流已经闭环。后续重点不应继续扩张基础页面，而应提高终端操作效率、自动化能力和故障诊断深度。

## 2. 竞品能力摘要

### 2.1 FinalShell

FinalShell 官方列出的差异化能力包括：

- 命令自动提示、智能匹配、历史和自定义参数。
- 在命令输入框中选择远端目录和文件。
- 终端、SFTP 同屏并同步切换目录。
- Ping 延迟/丢包、实时 Trace 路由和多地点 Ping。
- 进程管理器、监听端口与网络连接监控。
- 编辑器语法高亮、代码折叠、搜索和替换。
- Zmodem `rz/sz`、打包传输和自动压缩解压。
- 丰富终端配色、背景图片和连接加速。
- 云端同步及跨平台使用。

### 2.2 Termius

- IDE 风格命令自动补全。
- 加密 Vault、多设备同步和安全分享。
- 会话日志、工作区恢复和多人终端协作。
- SSH 密钥生成、FIDO2 硬件密钥和生物识别保护。
- 多平台 SSH、SFTP、端口转发和密钥管理。

### 2.3 SecureCRT

- 正则关键字高亮和可组合显示样式。
- 命令管理器、命令文件夹及多会话发送。
- Python 自动化、脚本录制和重复任务执行。
- 多级标签组、横向/纵向布局及工作区快照。
- X11、Agent 转发、智能卡认证。
- SFTP、Xmodem、Ymodem、Zmodem 和 Kermit。

### 2.4 Tabby

- 任意嵌套的横向和纵向拆分。
- 自动重连、X11 和 Agent 转发。
- Zmodem、终端输出保存和批量发送命令。
- 进程完成检测和桌面通知。
- 插件、主题和加密配置同步。
- SSH、Telnet、Serial 及本地 Shell。

### 2.5 iTerm2

- 粘贴历史、多行粘贴警告和高级粘贴。
- Copy Mode、即时回放、每行时间戳和全局搜索。
- 正则触发器、关键字高亮、自动响应和完成通知。
- Shell Integration，可识别 Prompt、命令、主机和当前目录。
- Smart Selection、注释、Badge 和捕获输出。

### 2.6 Warp

- 命令块和多线程工作流。
- AI 命令生成、错误解释和任务代理。
- 垂直会话、原生代码审查及云端/本地任务协同。
- 企业权限、秘密脱敏和密码管理器集成。

## 3. 功能缺口与优先级

### P0：终端日常效率

#### 3.1 智能命令栏

目标：把现有独立命令输入框升级为可发现、可补全、可安全执行的操作入口。

- 根据本地历史和快捷命令进行模糊补全。
- 根据当前远端目录补全文件和目录路径。
- 支持命令参数模板，例如 `systemctl restart {{service}}`。
- 参数模板执行前以表单收集值并展示最终命令预览。
- 支持键盘选择、补全、执行和取消。
- 敏感命令不进入历史；高风险命令执行前明确确认。

验收标准：

- 输入三个字符后 100 ms 内展示建议。
- 补全结果明确区分历史、快捷命令和远端路径。
- 路径包含空格、中文和单引号时能安全转义。
- 参数值不能改变模板结构或注入额外命令。

#### 3.2 会话日志与时间戳

- 用户可为单个会话开启或停止日志记录。
- 日志包含连接名称、开始/结束时间和可选逐行时间戳。
- 支持纯文本和结构化 JSONL 导出。
- 应用崩溃后已写日志保持可读。
- 默认关闭；启用前提示日志可能包含命令和秘密。
- 提供日志自动清理周期和最大占用限制。

验收标准：

- 1 MB 连续输出日志无缺失，终端输入不出现可感知延迟。
- 日志写入失败不影响 SSH 会话。
- 默认状态下不创建任何会话日志。

#### 3.3 关键字高亮、触发器和通知

- 用户可创建正则规则并配置前景色、背景色和粗体。
- 内置 `error`、`warning`、IPv4/IPv6 等可选规则。
- 匹配后可高亮、发 macOS 通知或仅记录事件。
- 支持终端 Bell、长任务完成和后台标签活动通知。
- 规则必须限制长度、复杂度和执行时间，避免正则拒绝服务。

验收标准：

- 1 MB/s 输出下高亮不阻塞终端渲染。
- 同一规则支持冷却时间，避免通知风暴。
- 系统通知只在用户明确授权后启用。

#### 3.4 多主机批量执行

- 从连接库选择多个 SSH 主机。
- 支持一次性批量命令和持续同步输入两种模式。
- 执行前显示目标主机、最终命令和风险提示。
- 每台主机独立显示状态、输出、耗时和退出码。
- 支持并发上限、取消、失败重试和仅重试失败项。
- 默认禁止在批量模式直接执行明显破坏性命令。

验收标准：

- 至少 20 个本地 SSH 夹具并发时 UI 保持响应。
- 单个主机失败不终止其他主机。
- 输出事件始终携带目标会话 ID，不发生串流混淆。

### P1：编辑、诊断和连接能力

#### 3.5 增强远端文本编辑器

建议使用 CodeMirror 6，保持按需加载。

- 行号、语法高亮、搜索替换和代码折叠。
- JSON/YAML 格式化、括号匹配和可选自动缩进。
- 保存冲突时显示本地、远端和基础版本 Diff。
- 保留现有 10 MB 限制、原子保存和修改时间检查。
- 支持切换到外部应用编辑并安全回传。

验收标准：

- 10 MB 上限按 UTF-8 字节计算。
- 搜索替换不会绕过远端修改冲突检查。
- 二进制和非法 UTF-8 文件仍拒绝内置编辑。

#### 3.6 进程管理与网络诊断

- 进程列表支持搜索、CPU/内存排序和完整命令行查看。
- 支持发送 `TERM`、`HUP`、`KILL`，操作前确认 PID 与命令。
- 展示监听端口、TCP/UDP 连接、状态和对应进程。
- 增加 Ping 延迟/丢包、Traceroute 和历史趋势。
- 缺少 `ss`、`lsof`、`traceroute` 或权限时局部降级。

验收标准：

- 进程操作不得通过用户交互 Shell 注入命令。
- PID 在确认后发生变化时拒绝操作，避免误杀新进程。
- 所有网络诊断均可取消并有执行超时。

#### 3.7 多级终端拆分

- 支持水平、垂直和嵌套拆分。
- 支持键盘在窗格间移动、调整尺寸和关闭窗格。
- 工作区恢复完整保存拆分树、比例和活动窗格。
- 单个会话不能同时绑定到两个可输入窗格。

验收标准：

- 4 个窗格下 resize 正确同步各自 PTY 尺寸。
- 关闭任意中间窗格后拆分树正确收拢。
- 小窗口下提供可恢复的最小尺寸策略。

#### 3.8 OpenSSH 配置与密钥管理

- 导入 `~/.ssh/config` 的 Host、Hostname、User、Port、IdentityFile、ProxyJump。
- 支持 Include，检测循环和越界路径。
- 生成 Ed25519 密钥并保存至用户选择位置。
- 显示公钥、SHA-256 指纹并支持复制。
- 可选一键部署公钥到远端 `authorized_keys`，执行前展示变更。
- 后续评估 SSH Certificate、FIDO2 和 Touch ID 设备绑定密钥。

验收标准：

- 导入不会覆盖已有连接，重复项必须由用户确认。
- 私钥内容不写入数据库、日志或导出文件。
- 公钥部署使用临时文件和权限检查，不破坏现有内容。

#### 3.9 终端安全与可用性

- 多行粘贴预览和高风险内容确认。
- 可选粘贴历史，默认只保存在内存。
- Copy Mode 和全局跨标签搜索。
- 终端输出行时间戳和活动标记。
- 后台任务完成通知。
- 可配置关键字最低对比度和光标增强。

### P2：高级协议和自动化

#### 3.10 Zmodem 与其他传输方式

- 识别 `rz/sz` 控制序列并启动 Zmodem 上传/下载。
- 文件仍通过原生选择器选择，禁止网页任意读取本地路径。
- 评估 SCP 作为 SFTP 不可用时的降级方案。
- X/Ymodem 和 Kermit 暂不实现，除非出现明确设备需求。

#### 3.11 Mosh、X11 与 Agent 转发

- Mosh 用于移动网络和频繁 IP 切换场景。
- X11 转发默认关闭，并明确提示本地 X Server 依赖。
- Agent 转发按连接独立开启，展示安全风险。
- 不在首轮实现 Telnet；Telnet 无加密，只在遗留设备需求明确时评估。

#### 3.12 自动化脚本

- 第一阶段提供受限任务编排，而不是直接开放任意插件权限。
- 支持顺序执行命令、等待匹配、超时、条件分支和文件传输。
- 提供运行预览、逐步日志、取消和失败恢复。
- 后续再评估 Python 脚本和录制键盘操作。

#### 3.13 可选加密同步

- 优先考虑用户自有 iCloud Drive、WebDAV 或 Git 仓库。
- 同步内容在本地加密后上传，服务端永远只看到密文。
- 冲突必须保留双方副本，不静默覆盖。
- 密码、私钥和主机信息分别提供同步开关。
- 不把账号系统和团队协作作为单机版前置条件。

### P3：探索性能力

#### 3.14 AI 辅助

- 自然语言生成命令时必须先展示命令和目标主机，禁止自动执行。
- 支持解释错误输出、生成只读诊断步骤和摘要会话日志。
- 默认不上传终端内容；使用云模型前必须明确选择范围并脱敏。
- 优先支持用户自带 API Key 或本地模型。

#### 3.15 插件体系与团队协作

- 完整插件系统会扩大权限和供应链风险，待核心 API 稳定后再设计。
- 多人终端、角色权限和审计日志属于团队版范围。
- 插件必须声明网络、文件、终端输入和凭据权限，默认全部拒绝。

## 4. 建议实施阶段

### 阶段 A：终端效率包

范围：3.1～3.4。

交付：智能命令栏、会话日志、关键字触发器、任务通知和多主机批量执行。

建议周期：3～5 周。

### 阶段 B：编辑与诊断包

范围：3.5～3.9。

交付：CodeMirror 编辑器、进程管理、端口/连接诊断、Ping/Trace、多级拆分、SSH Config 导入和密钥生成。

建议周期：4～6 周。

### 阶段 C：协议与自动化包

范围：3.10～3.13。

交付：Zmodem、Mosh/转发增强、受限任务编排和可选加密同步。

建议周期：4～7 周。

### 阶段 D：探索能力

范围：3.14～3.15。

AI、插件和团队协作必须在核心数据边界、权限模型和发布签名稳定后再开始。

## 5. 暂不优先投入

- 终端背景图片和大量装饰主题：收益低于命令效率和稳定性。
- 全球多地点 Ping：需要长期维护外部探测节点。
- SSH/RDP 商业加速：需要全球中继基础设施、合规和持续带宽成本。
- Telnet：协议无加密，仅为明确的遗留设备需求提供。
- 完整云账号与团队后台：偏离本地优先定位。
- 立即开放通用插件系统：会显著扩大安全边界和维护成本。
- AI 自动执行命令：风险不可接受，只允许生成、解释和人工确认。

## 6. 跨阶段工程要求

- 所有后台任务必须可取消、有超时并通过任务 ID 报告进度。
- 新功能不得把密码、私钥、完整命令或终端日志默认上传到网络。
- 批量命令、进程信号、公钥部署等高影响操作必须预览并确认。
- 终端输出处理不得阻塞输入，持续输出下仍需保持低延迟。
- 功能默认提供键盘操作和 VoiceOver 名称。
- 每个阶段完成后更新 `PLAN.md`、`CHANGELOG.md` 和 `docs/ACCEPTANCE.md`。
- 长测、大文件和跨设备验收继续独立执行，不纳入每次普通开发门禁。

## 7. 实施与验收状态

### 7.1 本轮常规验收

- `npm run lint`、`npm run build` 通过。
- P0/P1/Stage C 定向前端测试通过，包括智能命令、日志、高亮性能、拆分、编辑器、诊断、OpenSSH、协议设置和自动化预览。
- Rust 定向测试通过，包括 1 MB 日志、OpenSSH Include、自动化 Schema、协议探测、加密同步密文/冲突和 SCP 编译路径。
- 按用户要求未重跑 soak、1 GB 文件或 100k 文件测试；这些既有证据不冒充本轮执行结果。

### 7.2 后续仅在依赖具备时验收

1. 安装本机及远端 `lrzsz` 后实现并验证 Zmodem 控制序列接管、取消和二进制完整性。
2. 安装本机/远端 Mosh 并开放测试 UDP 端口后，验证漫游、IP 切换和恢复。
3. 安装 XQuartz 且更换/扩展为支持 X11 请求的 SSH 引擎后，验证可信主机逐连接转发。
4. P3 需先完成 Developer ID、公证、正式更新服务和权限/隐私设计评审。
